# Relational Sovereignty: The Participation Economy

**The Core Technical Proposition of PoliTickIt**

## 1. Defining Sovereignty

Unlike traditional social platforms that treat user data as a product, PoliTickIt treats user interaction as **Constituent Intelligence**.

## 2. The Sovereignty Ledger

All critical user interactions (Votes, Petitions, Representative Feedback) are stored in the local `participation_log`.

- **Verifiable**: The lineage of a vote can be traced from the device to the institutional dashboard without intermediary tampering.
- **ROI Reports**: B2B institutional users (Agencies, Representatives) see aggregate ROI on policy changes based on these deterministic participation signals.

## 3. The Consensus Ripple™

A graphical and mathematical model used to visualize the "velocity" of consensus within a district.

- **Calculation**: Weighted combination of `pulse_log` (interest) and `participation_log` (action).
- **Visualization**: Rendered as a dynamic heatmap or ripple effect in the **Nexus Dashboard**.
