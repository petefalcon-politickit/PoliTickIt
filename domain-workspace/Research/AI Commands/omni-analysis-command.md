# **AI Command Specification: OMNI-ANALYSIS**

- **Name**: Meta-Analytical Engine
- **Trigger**: `OMNI-ANALYSIS`
- **Version**: 1.1.0
- **Status**: Operational

## 🎯 Purpose

The `OMNI-ANALYSIS` command standardizes the "**Omni Analysis Standard**" process, transforming transient ideas into hardened, multi-pillar strategic blueprints.

## 📜 Execution Rules

### 1. Global Inheritance

- **Baseline**: Inherits all safety and precision standards from [Global AI Command Guardrails](global-command-guardrails.md).

### 2. The Omni Analysis Standard

Every analysis MUST conduct an **Evidence Grade Audit** (Marc Effron Standard) to establish the "Grade of Truth" for all claims:

- **High-Confidence**: Peer-reviewed meta-analysis or multi-study replication.
- **Medium-Confidence**: Organizational data or expert consensus.
- **Low-Confidence**: Popular press, case studies of single entities, or anecdotes.

### 3. The Quad-Pillar Standard

Every analysis MUST address:

1.  **Architecture**: Technical feasibility, complexity, and debt.
2.  **Intellectual Property**: Patentability, prior art risk, and claim anchors.
3.  **Market/Marketing**: Growth strategy, user segments, and retention logic.
4.  **Forensic Impact**: Clear "Success Metrics" and "Execution Roadmap."

### 4. Output Format

- **Location**: `documentation/[Context]/Analysis/[kebab-case-name].md`
- **Structure**: Abstract → Analysis (Quad-Pillar) → Final Recommendation.
- **Header**: Every blueprint MUST identify the **Author** as the individual providing the prompts/vision (e.g., Pete Falcon).

## 🛠️ Specialized Tasks

- [ ] **Thesis Extraction**: Clarify the core "Idea" and its intended impact.
- [ ] **Evidence Grade Audit**: Perform a Marc Effron research triage to identify the "Grade of Truth" for the proposal.
- [ ] **Pillar Pass**: Systematically audit the idea through Architecture, IP, and Marketing lenses using the **Omni Analysis Standard**.
- [ ] **Blueprint Drafting**: Author the final formal document following the Post-Grad standard.
