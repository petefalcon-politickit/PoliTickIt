# PoliManifestor™: Visionary Cheat Sheet

**Manifestation Keywords**: `MANIFEST` or `POLIFEST`
**Discovery Keyword**: `GENRE` or `POLIGENRE`
**Checkpoint Keyword**: `POLI-CHECK`
**Documentation Keyword**: `POLI-DOC`

---

## 🏗️ The Genre Catalog (Discovery)

Use `POLIGENRE [Name]` to see "Best-in-Class" blueprints for these categories:

| Genre            | Focus Area                                                              |
| :--------------- | :---------------------------------------------------------------------- |
| **Ethics**       | Ethics Committee findings, misconduct tracking, and integrity pivots.   |
| **Fiscal**       | Tax implications in a district, budget allocation, and local ROI.       |
| **Velocity**     | Legislative speed, floor activity intensity, and productivity alerts.   |
| **Finance**      | Campaign finance correlations, PAC donation pivots, and lobbyist ties.  |
| **Sentiment**    | Constituent sentiment alignment and district-wide consensus ripples.    |
| **Audit**        | Institutional ROI, performance grading, and alignment Matrix results.   |
| **Productivity** | Legislative velocity, bill sponsorship frequency, and floor efficiency. |

## 📋 Prompt Structure

### Standard

`POLIFEST [Classifiers] : [The Story]`

### Blueprint Reference (Post-POLIGENRE)

If you've just run `POLIGENRE`, you can manifest a blueprint directly without re-typing the story:
`POLIFEST [Blueprint Name or #] [Classifiers]`

## 📍 Classifiers (Scope Control)

Use these to constrain the "Truth" being manifested. All snaps are generalized into **Classes** (stored in `libs/contracts/snap-classes/`) by default to ensure they scale across all representatives and policies.

| Category   | Keys                                                              | Examples                                                      |
| :--------- | :---------------------------------------------------------------- | :------------------------------------------------------------ |
| **Region** | `region:city`, `region:county`, `region:state`, `region:national` | `region:state:tx`, `region:national:us`, `region:city:austin` |
| **Policy** | `policy:[area]`                                                   | `policy:healthcare`, `policy:energy`, `policy:defense`        |
| **Party**  | `party:[code]`                                                    | `party:dem`, `party:rep`, `party:ind`                         |
| **Tier**   | `tier:[level]`                                                    | `tier:standard`, `tier:intelligence`, `tier:roi-auditor`      |

## 🛡️ The Visionary's Choice (The Gate)

1.  **Use Case Presentation**: Review the business-centric value and user impact summary.
2.  **PoliFest Preview**: Review the technical analysis summary (ready for the backlog if deprioritized).
3.  **Blueprint**: Review the data sources (Official APIs/Batch only).
4.  **Data Staging**: Review the technical write-up for "Always Available Content" if batch data is used.
5.  **Visual Audit**: Use the [Validation Cheatsheet](VALIDATION_CHEAT_SHEET.md) to check Vertical Rhythm, Weights, and Alignments.
6.  **Tracks**: If asked, select Track A, B, or C.
7.  **Execute**: Say "Execute" to push code to the .NET Registry and Mobile Ledger.
8.  **Priority**: Look for the "Next Best Action" at the end of the sprint.

## 🏁 The Checkpoint (The Completion)

When your feature is validated, run:

> **POLI-CHECK**

- **Documentation Sync**: Updates the Registry, Backlog, and Technical Docs.
- **Staging Commit**: Finalizes Universal Ingestion manifests.
- **Priority Roadmap**: Provides your next 2-3 most impactful sprint items.
